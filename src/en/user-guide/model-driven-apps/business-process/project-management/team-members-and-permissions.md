# Project Team Members and Permissions

Not everyone in a Project has the same access — and each company keeps their own Roles. Some will have access to read and write in the entire Project, some only for a certain Stage, while others will only see Tasks assigned to them.

Additionally, while companies may have long-term Teams, they can onboard other members temporarily — either to balance workload or as a substitute for a missing member.

## Adding Members to a Project

After creating a Project, the next most important step is selecting its members. For each member, you specify:

| Field | Required | Description |
|:------|:---------|:------------|
| User | Yes | The person being added to the project |
| Role | Yes | Their role within the project |
| Stage | No | Restricts access to a specific project stage |
| From | Yes | The date they gain access |
| To | No | The date they lose access |

This also helps keep track of who participated in the past.

## Assigning Tasks to Non-Members

When assigning Tasks to members (as accountable or responsible), if the user is not yet part of the Project, the person assigning will be prompted to confirm their role and the dates for access before the assignment is saved.
